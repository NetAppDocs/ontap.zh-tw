= 使用 Upgrade Advisor 或 Upgrade Health Checker 為 ONTAP 升級做好準備
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
您可以使用 Active IQ Digital Advisor 中的 Upgrade Advisor 或 Upgrade Health Checker 自動產生專屬於您環境和叢集組態的升級報告。這兩種工具都能提供智慧資訊，協助您規劃升級、識別阻礙因素和警告，並將不確定性和風險降至最低。



== 準備使用 Upgrade Advisor 升級

Upgrade Advisor 可識別環境中可透過升級至較新版本的 ONTAP 來解決的問題，並提供一份報告，列出您在升級至的 ONTAP 版本中可能需要注意的問題。Upgrade Advisor 還會提供建議的升級路徑，並產生專門針對您叢集組態的升級計畫。

Upgrade Advisor 特別適合環境較小且偏好以雲端為基礎、以 UI 為中心體驗的使用者。

.開始之前
* 使用 Upgrade Advisor 需要有效的 link:https://www.netapp.com/us/services/support-edge.aspx["部門服務SupportEdge"^] link:https://docs.netapp.com/us-en/active-iq/upgrade_advisor_overview.html["數位顧問"^] 合約。
* AutoSupport 必須啟用，Upgrade Advisor 才能自動存取產生升級計畫所需的 AutoSupport 記錄。如果您尚未啟用 AutoSupport，可以 https://kb.netapp.com/on-prem/ontap/Ontap_OS/OS-KBs/How_to_manually_upload_AutoSupport_messages_to_NetApp_in_ONTAP_9["手動上傳 AutoSupport 檔案"^]。


.步驟
. https://aiq.netapp.com/["啟動 Active IQ 數位顧問"^]
. 在 Digital Advisor 中link:https://docs.netapp.com/us-en/active-iq/task_view_risk_and_take_action.html["檢視與叢集相關的任何風險、並手動採取修正行動"^]。
+
執行 ONTAP 升級之前、 * 軟體組態變更 * 、 * 硬體組態變更 * 和 * 硬體更換 * 類別所包含的風險必須先解決。

. 檢閱建議的升級路徑和 link:https://docs.netapp.com/us-en/active-iq/upgrade_advisor_overview.html["產生您的升級計畫"^]。


.下一步
* 您應該檢閱 link:../release-notes/index.html["發行說明ONTAP"] 對於 Upgrade Advisor 建議用於叢集的目標 ONTAP 版本、您應該遵循 Upgrade Advisor 所產生的計畫來升級叢集。
* 您應該 link:reboot-sp-bmc.html["重新啟動 SP 或 BMC"] 升級開始之前。




== 準備使用 Upgrade Health Checker 進行升級

Upgrade Health Checker 是一款本地部署工具，適用於擁有大型或複雜環境、網路存取受限或已部署自動化和編排框架的使用者。使用 Upgrade Health Checker 無需簽訂 SupportEdge 合約。

.開始之前
您必須先link:https://docs.netapp.com/us-en/upgrade-health-checker/set-up-upgrade-health-checker.html["下載並設定 Upgrade Health Checker"]才能使用它來產生升級計劃。

.步驟
. link:https://docs.netapp.com/us-en/upgrade-health-checker/generate-report-upgrade-health-checker.html["使用 Upgrade Health Checker 產生升級計畫"^].


.下一步
* 您應該查看link:../release-notes/index.html["發行說明ONTAP"]針對叢集建議的目標 ONTAP 版本；然後，您應該按照 Upgrade Health Checker 產生的計劃來升級叢集。
* 您應該 link:reboot-sp-bmc.html["重新啟動 SP 或 BMC"] 升級開始之前。
