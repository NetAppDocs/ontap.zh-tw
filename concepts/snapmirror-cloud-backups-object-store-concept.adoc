= SnapMirror 雲端備份至物件儲存
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SnapMirror Cloud 是一種備份和復原技術，專為希望將其資料保護工作流程遷移到雲端或本地物件儲存解決方案的 ONTAP 使用者而設計。正在擺脫傳統磁帶備份架構的組織可以使用物件儲存作為長期資料保留和歸檔的替代儲存庫。SnapMirror Cloud 提供 ONTAP 到物件儲存的複製功能，作為永久增量備份策略的一部分。

SnapMirror Cloud 複製是 ONTAP 的一項授權功能。SnapMirror Cloud 於 ONTAP 9.8 中推出，作為 SnapMirror 複製技術系列的延伸。雖然 SnapMirror 常用於 ONTAP 對 ONTAP 的備份，但 SnapMirror Cloud 使用相同的複製引擎，將快照從 ONTAP 傳輸至符合 S3 標準的物件儲存備份。

SnapMirror Cloud 專為備份應用場景而設計，支援長期保留和歸檔工作流程。與 SnapMirror 類似，初始 SnapMirror Cloud 備份會執行磁碟區的基線傳輸。對於後續備份，SnapMirror Cloud 會產生來源磁碟區的快照，並將僅包含已變更資料區塊的快照傳輸到物件儲存目標。

SnapMirror Cloud 關係可在 ONTAP 系統與選取的內部部署及公有雲物件儲存目標之間進行設定，包括 Amazon S3、Google Cloud Storage 和 Microsoft Azure Blob Storage。其他內部部署物件儲存目標包括 StorageGRID 和 ONTAP S3。

除了使用 ONTAP System Manager 管理 SnapMirror Cloud 組態外，還有幾種協調選項可用於管理 SnapMirror Cloud 備份：

* 多家第三方備份合作夥伴提供 SnapMirror 雲端複製支援。參與的供應商可在 link:https://www.netapp.com/blog/new-backup-architecture-snapdiff-v3/["NetApp部落格"^] 找到。
* NetApp Backup and Recovery 是適用於ONTAP環境的NetApp原生解決方案
* API可用於開發自訂的資料保護工作流程軟體、或是運用自動化工具



NOTE: 由 System Manager 管理的 SnapMirror Cloud 複寫僅支援 StorageGRID 和 ONTAP S3 作為物件儲存目的地。不支援其他提供者。


NOTE: SnapMirror 雲端 API 的使用通常僅限於備份合作夥伴。希望開發自有 API 解決方案的客戶應聯絡其 NetApp 客戶經理團隊，以了解授權條款。
