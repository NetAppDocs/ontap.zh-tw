= 主磁碟機當機時、磁碟區移動作業會卡住
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


.問題：
[role="lead"]
當主要站台處於SMBC關係中當主站台當機時、Volume Move作業會無限期地停留在轉換延後狀態。
當主要站台當機時、次要站台會執行自動非計畫性容錯移轉（AUFO）。當磁碟區移動作業正在進行時、當AUFO觸發時、磁碟區移動就會卡住。

.解決方案：
中止卡住的Volume Move執行個體、然後重新啟動Volume Move作業。
