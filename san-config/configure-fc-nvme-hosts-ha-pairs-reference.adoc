= 學習如何使用 ONTAP 系統設定 FC 或 FC-NVMe 光纖架構以及直接連接配置
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
對於 FC 和 FC/NVMe，儲存解決方案和主機之間的網路拓撲稱為光纖通道網路（或稱為交換矩陣）。為了建立冗餘以防止資料存取中斷，建議您使用多重交換矩陣配置，在 SAN 中設定高可用性（HA）配對，即使用兩個或多個交換器。主機和儲存之間僅使用一個交換器的單交換矩陣配置不具備完全冗餘性。

從 ONTAP 9.19.1 開始，您也可以將 FC 或 FC-NVMe SAN 主機直接連接到 HA 配對，而無需使用交換器。直接連接組態在遠端站台和混合式環境中尤其有價值，因為在這些環境中，簡單性至關重要。

級聯、部分網狀、全網狀、核心-邊緣和導向器架構都是將 FC 交換器連接到架構的業界標準方法，並且都受支援。除嵌入式刀鋒交換器外，不支援使用異質 FC 交換器架構。特定例外情況列於 link:https://imt.netapp.com/matrix/["互通性對照表工具"^]。架構可以包含一個或多個交換器，儲存控制器可以連接到多個交換器。

使用不同作業系統（例如 Windows、Linux 或 UNIX）的多台主機可以同時存取儲存控制器。主機需要安裝並配置受支援的多路徑解決方案。支援的作業系統和多路徑解決方案可透過互通性矩陣工具進行驗證。



== Multifabric FC 和 FC-NVMe 組態

建議您使用 HA 配對和至少兩台交換器來設定 FC 和 FC-NVMe SAN 主機。[EH5.1][FD5.2][EH5.3]這可在網路架構和儲存系統層提供備援功能，以支援容錯和不中斷營運。

在多重交換矩陣 HA 配對組態中，兩個或多個交換器將 HA 配對連接至一個或多個主機。每個交換器會建立一個獨立的交換矩陣，其中包含主機與 HA 配對之間通過該交換器的路徑。新增交換器會建立額外的交換矩陣。多重交換矩陣組態可包含兩個或多個交換器及其對應的交換矩陣。為簡化說明，下圖僅顯示兩個交換器，代表兩個獨立的交換矩陣。

圖中的 FC 目標連接埠編號（ 0c 、 0d 、 1a 、 1b ）為範例。實際的連接埠編號會因儲存節點的機型和是否使用擴充介面卡而有所不同。

image:scrn_en_drw_fc-32xx-multi-HA.png["Mertifabric HA配對圖"]



== 單一架構 FC 和 FC-NVMe 組態

在單一網狀架構 HA 配對組態中，一台交換器將 HA 配對連接到一個或多個主機。單一交換器會建立單一網狀架構，其中包含主機與 HA 配對之間的所有路徑。由於沒有備援，交換器故障可能會導致服務中斷。

圖中的 FC 目標連接埠編號（ 0A 、 0c ）為範例。實際的連接埠編號會因儲存節點的機型和是否使用擴充介面卡而有所不同。

所有支援 FC 組態的平台都支援單一架構 HA 配對組態。

image:scrn_en_drw_fc-62xx-single-HA.png["單一交換器架構HA配對圖"]



== 直接連接式 FC 和 FC-NVMe 組態

從 ONTAP 9.19.1 開始，AFF、ASA、ASA r2 和 FAS 系統可使用直接連接 FC 和 FC-NVMe 組態。在直接連接組態中，一或多個主機可直接連接至 HA 配對，而不需要交換器。此組態在遠端站台和混合式環境中特別有價值，因為簡單性是關鍵。

image:scrn_en_drw_fc-302020-direct-sing-on.png["直接連接組態"]
